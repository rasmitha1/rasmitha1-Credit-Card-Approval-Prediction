Credit Card Approval Prediction
A machine learning web app that predicts whether a credit card application is likely to be Approved or Rejected, based on applicant details such as income, employment, family status, and housing.
Overview
This project combines a data science workflow (Jupyter Notebook) with a Flask web application:
Data preprocessing & model training — cleaning, merging, and encoding applicant data, then training and comparing multiple classification models.
Web app — a simple form where a user enters applicant details and receives an instant Approved/Rejected prediction.

Dataset
Built from two datasets:
Application record — applicant details (gender, income, education, occupation, housing, etc.)
Credit record — monthly credit history/payment status, used to derive the target label
The two are merged on applicant `ID`, and a binary `TARGET` label is derived from the credit history (1 = risky/likely default, 0 = good standing).
Approach
Data cleaning — removed duplicates, handled missing values, derived `AGE` and `YEARS_EMPLOYED` from raw day counts.
Feature encoding — categorical columns (gender, income type, education, family status, housing type, occupation, etc.) encoded using `LabelEncoder`.
Note on feature selection — `open_month`, `end_month`, and `window` (derived directly from credit history records) were excluded from the final model, since they are closely tied to how the target label itself was constructed and would not be realistically available for a brand-new applicant. The final model is trained only on demographic and application-level features.
Model training — multiple classifiers were trained and compared; the best-performing model was saved for use in the web app.
Decision threshold — since the dataset is imbalanced (most applicants are "good"), the model's predicted probability of risk is compared against a tuned threshold (rather than the default 0.5 midpoint) to produce the final Approved/Rejected decision.
Running the App
(Optional) Create and activate a virtual environment.
Install dependencies:
```
   pip install -r requirements.txt
   ```
Make sure `models/credit_card_model.pkl` and `models/encoders.pkl` exist. These are generated by running `notebooks/Credit_Card_Approval_Final.ipynb` end-to-end (Restart & Run All), which saves both files into `models/`.
Start the app:
```
   python app.py
   ```
Open a browser to `http://127.0.0.1:5000`, fill in the applicant form, and submit to see the prediction.
Limitations
The dataset's target label is derived from actual credit bureau history, while the web form only collects demographic/application information. Since credit-history features were intentionally excluded to avoid leakage, prediction accuracy is inherently limited compared to models that include real payment history.
This project is for educational/demonstration purposes and should not be used for real credit decisions.
